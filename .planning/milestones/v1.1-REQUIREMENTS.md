## v1.1 Requirements

Requirements for milestone v1.1. Each maps to roadmap phases.

### Thread Pool (SYNC)

- [x] **SYNC-01**: Sync handlers run on Rayon work-stealing pool, not blocking Tokio poll cycle
- [x] **SYNC-02**: `ConsumerConfigBuilder::rayon_pool_size(u32)` for configurable pool size
- [x] **SYNC-03**: Graceful shutdown coordinates with Rayon drain (30s timeout)

### Async Timeout (TMOUT)

- [x] **TMOUT-01**: `@handler(topic, timeout=X)` Python API — wire existing `invoke_mode_with_timeout`
- [x] **TMOUT-02**: Timeout metadata propagated to DLQ envelope (timeout_duration, last_processed_offset)
- [x] **TMOUT-03**: Timeout metric (count per handler) emitted to Prometheus

### Handler Middleware (MIDW)

- [x] **MIDW-01**: `HandlerMiddleware` trait in Rust with before/after/on_error hooks
- [x] **MIDW-02**: Built-in logging middleware (span events on handler start/complete/error)
- [x] **MIDW-03**: Built-in metrics middleware (latency histogram, throughput counter)
- [x] **MIDW-04**: Python API `@handler(middleware=[Logging(), Metrics()])` per handler

### Streaming Handler (STRM)

- [x] **STRM-01**: `HandlerMode::StreamingAsync` for persistent async iterable handlers
- [x] **STRM-02**: `@stream_handler(topic)` Python API — long-lived handler
- [x] **STRM-03**: Lifecycle management (start/subscribe, run/loop, stop/drain, error recovery)
- [x] **STRM-04**: Per-stream backpressure propagation
