# Requirements: KafPy v1.8 — Public API Foundation

**Milestone:** v1.8
**Goal:** Clean, Pythonic public API and packaging — Rust internals hidden, instance-based config, decorator + explicit handler registration, meaningful Python exceptions, docs/guides foundation.

---

## API Design (API-##)

- [ ] **API-01**: All public Python classes use `PascalCase` naming; methods use `snake_case`. Type hints on all public signatures.
- [ ] **API-02**: No global mutable state — all runtime state held in instance objects (e.g., `Consumer` instance).
- [ ] **API-03**: Rust internals are private by default; only explicitly marked `pub` items are accessible from Python.
- [ ] **API-04**: `__all__` defined in every public Python module listing only the intended public API.
- [ ] **API-05**: Public API is stable — no underscore-prefixed items in `__all__`, no raw Rust struct names exposed.

## Handler Registration (REG-##)

- [ ] **REG-01**: Decorator-based registration: `@consumer.handler(topic="...", routing=...)` annotates a callable as a handler.
- [ ] **REG-02**: Explicit registration: `consumer.register_handler(topic, handler_fn, routing=...)` as an alternative API.
- [ ] **REG-03**: Handler callable signature: `def handler(msg: KafkaMessage, ctx: HandlerContext) -> HandlerResult`.
- [ ] **REG-04**: Sync, async, and batch handlers all use the same registration API — differentiated by callable type (plain fn, async fn, generator fn).
- [ ] **REG-05**: `KafPy` (or `ConsumerApp`) is the main runtime object owning worker pool, dispatcher, and coordinator lifecycle.
- [ ] **REG-06**: `app.start()` begins consuming; `app.stop()` initiates graceful drain and shutdown.
- [ ] **REG-07**: `app.run()` convenience method: `start()` + block until `stop()` signal.

## Configuration Model (CFG-##)

- [ ] **CFG-01**: `ConsumerConfig` object: bootstrap_servers, group_id, topics, auto_offset_reset, enable_auto_commit, ...
- [ ] **CFG-02**: `RoutingConfig` object: routing_mode (pattern/header/key/python/default), fallback_handler.
- [ ] **CFG-03**: `RetryConfig` object: max_attempts, base_delay, max_delay, jitter_factor.
- [ ] **CFG-04**: `BatchConfig` object: max_batch_size, max_batch_timeout_ms.
- [ ] **CFG-05**: `ConcurrencyConfig` object: num_workers (per handler).
- [ ] **CFG-06**: Config objects are immutable builders or frozen after construction; no runtime mutation.
- [ ] **CFG-07**: `ConsumerConfig` passed to `KafPy()` constructor; per-handler config passed at registration time.

## Error Handling (ERR-##)

- [ ] **ERR-01**: Python-side exceptions in `kafpy.exceptions`: `KafPyError`, `ConsumerError`, `HandlerError`, `ConfigurationError`.
- [ ] **ERR-02**: `KafPyError` is the base; framework errors inherit from it, not from `Exception` directly.
- [ ] **ERR-03**: Rust errors translated to Python exceptions at the PyO3 boundary with context-preserving messages.
- [ ] **ERR-04**: `KafkaMessage` access errors (missing key, wrong type) raise `HandlerError`, not raw Rust panics.
- [ ] **ERR-05**: `kafpy.exceptions` is the only public import path for exceptions; nothing from `_kafpy` or `kafpy._rust` leaks.

## Package Structure (PKG-##)

- [ ] **PKG-01**: Top-level `import kafpy` exposes: `KafPy`, `ConsumerConfig`, `RoutingConfig`, `RetryConfig`, `BatchConfig`, `ConcurrencyConfig`, `KafkaMessage`, `HandlerContext`, `HandlerResult`, `KafPyError`, `ConsumerError`, `HandlerError`, `ConfigurationError`.
- [ ] **PKG-02**: `kafpy/exceptions.py` — all public exception classes.
- [ ] **PKG-03**: `kafpy/config.py` — all public config classes.
- [ ] **PKG-04**: `kafpy/consumer.py` — `KafPy` class.
- [ ] **PKG-05**: `kafpy/handlers.py` — `KafkaMessage`, `HandlerContext`, `HandlerResult`, registration decorators.
- [ ] **PKG-06**: `kafpy/constants.py` — routing mode enums, handler mode strings (if needed at Python level).
- [ ] **PKG-07**: `kafpy/_private.py` — private internals NOT in `__all__`, excluded from docs.
- [ ] **PKG-08**: `pyproject.toml` or `setup.py` configured for maturin with correct package metadata.

## Documentation (DOC-##)

- [ ] **DOC-01**: `README.md` at repo root with: install instructions, quickstart example (< 30 lines), feature list.
- [ ] **DOC-02**: `docs/` directory with at minimum: quickstart guide, API reference (auto-generated from docstrings).
- [ ] **DOC-03**: `kafpy/guides/` module as Python package with `.md` files: getting-started.md, handler-patterns.md, configuration.md, error-handling.md.
- [ ] **DOC-04**: Every public class and function has a docstring: purpose, args, returns, raises.
- [ ] **DOC-05**: Quickstart example demonstrates: install, config, handler decorator, `app.run()`.
- [ ] **DOC-06**: `docs/guides/handler-patterns.md` covers: sync handler, async handler, batch handler, error handling in handlers.

## Packaging (BUILD-##)

- [ ] **BUILD-01**: `pyproject.toml` with `build-system = { backend = "maturin" }` and `project.dependencies` including `rdkafka` system library notes.
- [ ] **BUILD-02**: `Cargo.toml` in project root (for maturin) exposes `kafpy-python` as the Python extension module name.
- [ ] **BUILD-03**: `kafpy/__init__.py` does `from .consumer import KafPy` and `from .config import *` etc.; no star-import of internal modules.
- [ ] **BUILD-04**: `maturin develop` works without additional configuration on a machine with `librdkafka` installed.
