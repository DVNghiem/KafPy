# Requirements — Milestone v1.5

**Goal:** Extensible routing with pattern/header/key matching and optional Python fallback. Rust is fast-path owner. Python routing is optional and GIL-limited.

---

## v1.5 Requirements

### Routing Core

- [ ] **ROUTER-01**: `RoutingContext` struct — holds topic, headers, key, payload (zero-copy reference, no copies)
- [ ] **ROUTER-02**: `RoutingDecision` enum — `Route(handler_id)`, `Drop`, `Reject(reason)`, `Defer`
- [ ] **ROUTER-03**: `Router` trait — `route(&self, ctx: &RoutingContext) -> RoutingDecision`
- [ ] **ROUTER-04**: `TopicPatternRouter` — glob/regex pattern matching on topic name
- [ ] **ROUTER-05**: `HeaderRouter` — route by header key presence or value match
- [ ] **ROUTER-06**: `KeyRouter` — route by message key prefix/exact match
- [ ] **ROUTER-07**: `RoutingChain` — chains routers with precedence: pattern → header → key → python → default

### Python Integration

- [ ] **PYROUTER-01**: `PythonRouter` — `Py<PyAny>` callback, called only on prior routers returning `Defer`
- [ ] **PYROUTER-02**: `spawn_blocking` for GIL-limited Python routing (non-blocking fast path)
- [ ] **PYROUTER-03**: Python router returns `RoutingDecision` via `Py<PyAny>` callback

### Dispatcher Integration

- [ ] **DISPATCH-01**: Router injected into `ConsumerDispatcher`, called before queue selection
- [ ] **DISPATCH-02**: `RoutingDecision::Drop` advances offset (no-op), `Reject` → DLQ, `Defer` → default handler

### Configuration

- [ ] **CONFIG-01**: `ConsumerConfig` exposes `routing_rules: Vec<RoutingRule>`
- [ ] **CONFIG-02**: `RoutingRule` — pattern/header/key config + target handler + priority

---

## Out of Scope

- Schema-based routing (Avro/Protobuf) — requires schema registry, deferred
- Content-based routing (payload parsing) — Python fallback only
- Multi-handler fan-out (single handler per message) — deferred
- Business logic inside routing path — Rust routing is pure decision, no business logic
