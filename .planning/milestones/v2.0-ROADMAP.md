# Roadmap: KafPy v2.0 Code Quality Refactor

**Milestone:** v2.0
**Goal:** Improve code quality without changing behavior — smaller focused modules, clear boundaries, less duplication, fewer god objects, explicit state models, cleaner naming, lower coupling, high cohesion.
**Granularity:** Standard (6 phases)
**Coverage:** 31/31 requirements mapped (100%)

---

## Phases

- [x] **Phase 1: Extract Duplicated Logic** — Extract handle_execution_failure(), message_to_pydict(), flush_partition_batch() helpers; verify NoopSink duplication
- [ ] **Phase 2: Split worker_pool/** — Extract PartitionAccumulator to batch/, split worker_loop/batch_worker_loop into separate files
- [ ] **Phase 3: Split dispatcher/ + Extract runtime/** — Extract ConsumerDispatcher, extract RuntimeBuilder to runtime/ module
- [ ] **Phase 4: State Machine Extraction** — Replace implicit Option/bool state with explicit WorkerState/BatchState/ShutdownState/RetryState enums
- [ ] **Phase 5: Split coordinator/** — Split OffsetTracker/OffsetCommitter to offset/, ShutdownCoordinator to shutdown/, RetryCoordinator to retry/
- [ ] **Phase 6: Type Safety** — HandlerId newtype, unified error module, Send+Sync verification

## Phase Details

### Phase 1: Extract Duplicated Logic

**Goal:** Eliminate duplication patterns in worker_pool/mod.rs and python/handler.rs without changing behavior

**Depends on:** Nothing

**Requirements:** DUP-01, DUP-02, DUP-03, DUP-04

**Success Criteria** (what must be TRUE):
1. `handle_execution_failure()` helper function exists and handles both Error and Rejected execution paths identically
2. `message_to_pydict()` helper exists in python/handler.rs and is called from invoke(), invoke_batch(), and invoke_async()
3. `flush_partition_batch()` helper consolidates all 6 batch flush repetitions in batch_worker_loop
4. NoopSink duplication verified — if identical to observability NoopSink, consolidated into shared location
5. All GATES pass: cargo clippy clean, cargo test --lib passes, no API changes

**Plans:** 1 plan
- [x] 01-01-PLAN.md — Extract duplicated helper functions (handle_execution_failure, message_to_pydict, flush_partition_batch, NoopSink consolidation)

### Phase 2: Split worker_pool/

**Goal:** Split the 1309-line worker_pool/mod.rs into focused modules by responsibility

**Depends on:** Phase 1

**Requirements:** SPLIT-A-01, SPLIT-A-02, SPLIT-A-03, SPLIT-A-04, SPLIT-A-05, SPLIT-A-06

**Success Criteria** (what must be TRUE):
1. PartitionAccumulator extracted to worker_pool/accumulator.rs (new batch/ module target)
2. batch_worker_loop() extracted to worker_pool/batch_loop.rs
3. worker_loop() extracted to worker_pool/worker.rs
4. WorkerPool struct extracted to worker_pool/pool.rs
5. PartitionAccumulator renamed to clarify its general-purpose nature (e.g., PerPartitionBuffer)
6. BatchAccumulator moved to python/batch.rs (handler-mode specific)
7. worker_pool/mod.rs becomes thin re-export layer
8. All GATES pass: cargo clippy clean, cargo test --lib passes, no API changes

**Plans:** 2 plans in 2 waves
- [ ] 02-01-PLAN.md — Extract PerPartitionBuffer and BatchAccumulator data structures (Wave 1)
- [ ] 02-02-PLAN.md — Extract worker_loop, batch_worker_loop, and WorkerPool (Wave 2)

### Phase 3: Split dispatcher/ + Extract runtime/

**Goal:** Split dispatcher/mod.rs and extract RuntimeBuilder from pyconsumer.rs

**Depends on:** Phase 2

**Requirements:** SPLIT-B-01, SPLIT-B-02, SPLIT-B-03, SPLIT-B-04

**Success Criteria** (what must be TRUE):
1. ConsumerDispatcher extracted from dispatcher/mod.rs to dispatcher/consumer_dispatcher.rs
2. RuntimeBuilder extracted from pyconsumer.rs to runtime/mod.rs with new runtime/ module
3. PyO3 integration verified with new RuntimeBuilder pattern (compile and basic test)
4. Runtime assembly order preserved: config → tracker → dispatcher → handler → worker → committer
5. pyconsumer.rs becomes thin PyO3 wrapper (~60 lines target)
6. All GATES pass: cargo clippy clean, cargo test --lib passes, no API changes

### Phase 4: State Machine Extraction

**Goal:** Replace implicit Option/bool state machines with explicit compiler-verified enums

**Depends on:** Phase 3

**Requirements:** STATE-01, STATE-02, STATE-03, STATE-04, STATE-05

**Success Criteria** (what must be TRUE):
1. WorkerState enum replaces `Option<OwnedMessage>` in worker_loop — compiler enforces exhaustiveness
2. BatchState enum replaces `backpressure_active: bool` in batch_worker_loop
3. ShutdownState enum replaces bool-like ShutdownPhase struct (follows PartitionState pattern)
4. RetryState enum replaces bool-like RetryCoordinator state fields
5. All state enum variants have exhaustive match arms — no wildcard `_` patterns remain
6. All GATES pass: cargo clippy clean, cargo test --lib passes, no API changes

**Plans:** 1 plan
- [ ] 01-01-PLAN.md — Extract duplicated helper functions (handle_execution_failure, message_to_pydict, flush_partition_batch, NoopSink consolidation)
