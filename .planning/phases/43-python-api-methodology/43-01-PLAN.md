   - `run_scenario_py(scenario_name: &str, config_json: &str) -> Py<PyDict>` — deserializes scenario from JSON, calls `BenchmarkRunner::new(None).run_scenario()`, returns result dict with all fields from BenchmarkResult. On error, raises PyErr.
   - `run_hardening_checks_py(result_json: &str) -> Py<PyList>` — deserializes BenchmarkResult from JSON, calls `HardeningRunner::run_all()`, returns list of ValidationResult dicts. On error, raises PyErr.

2. Add `m.add_wrapped!(pyo3::wrap_pyfunction_bindings!(run_scenario_py))?;` and `m.add_wrapped!(pyo3::wrap_pyfunction_bindings!(run_hardening_checks_py))?;` to the _kafpy pymodule init function.

Key implementation notes:
- Use pyo3::prelude::* already imported in lib.rs
- run_scenario_py: deserialize scenario using serde_json from &config_json, create BenchmarkRunner::new(None), await run_scenario(), then build a PyDict with all result fields including scenario_name from result.scenario_config.scenario_name
- run_hardening_checks_py: deserialize BenchmarkResult from result_json using serde_json, call HardeningRunner::run_all(), build a Vec of PyDict entries (one per ValidationResult), return as PyList
- Both functions must use `py: pyo3::Python<'_>` as first arg for pyfunction signature
- Import BenchmarkRunner, HardeningRunner, ValidationResult from crate::benchmark
- Import serde_json for JSON deserialization
- Keep the benchmark module pub(crate) — only the pyfunction wrappers are public Python API

3. Define frozen dataclasses (PY-02, PY-03, PY-04):

   ```python
   from dataclasses import dataclass, field
   from typing import Any, Optional

   @dataclass(frozen=True)
   class ScenarioConfig:
       """Scenario configuration for a benchmark run (mirrors Rust ScenarioConfig)."""
       scenario_type: str
       num_messages: int
       payload_bytes: int
       rate: Optional[int]  # messages per second, None = unlimited
       warmup_messages: int
       failure_rate: float

   @dataclass(frozen=True)
   class BenchmarkResult:
       """Immutable benchmark result returned after a scenario run."""
       scenario_config: ScenarioConfig
       total_messages: int
       duration_ms: int
       throughput_msg_s: float
       latency_p50_ms: float
       latency_p95_ms: float
       latency_p99_ms: float
       error_rate: float
       memory_delta_bytes: int
       timestamp_ms: int

   @dataclass(frozen=True)
   class BenchmarkReport:
       """Human-readable benchmark report with metrics and validation results."""
       scenario_name: str
       metrics: dict[str, Any]
       passed_checks: list[str]
       suggestions: list[str]

   @dataclass(frozen=True)
   class ValidationResult:
       """Result of a single hardening check."""
       check_name: str
       passed: bool
       details: str
       suggestions: list[str]
   ```

4. Define the public `run_scenario` function (PY-01):
   ```python
   def run_scenario(
       scenario_type: str,
       *,
       num_messages: int = 100_000,
       payload_bytes: int = 256,
       rate: Optional[int] = None,
       warmup_messages: int = 1000,
       failure_rate: float = 0.0,
       target_topic: str = "benchmark",
   ) -> BenchmarkResult:
       """
       Run a benchmark scenario and return the result.

       Args:
           scenario_type: One of "throughput", "latency", "failure", "batch_vs_sync", "async_vs_sync"
           num_messages: Total messages to send
           payload_bytes: Size of each message payload
           rate: Target messages per second (None = unlimited)
           warmup_messages: Warmup messages excluded from measurement
           failure_rate: Failure injection rate (0.0 to 1.0)
           target_topic: Kafka topic to use

       Returns:
           BenchmarkResult with latency/throughput metrics
       """
   ```

   Implementation:
   - Build a config dict mirroring ThroughputScenario/LatencyScenario fields from src/benchmark/scenarios.rs
   - Serialize to JSON and call `run_scenario_py(scenario_type, json.dumps(config))`
   - Convert the returned dict to a BenchmarkResult instance
