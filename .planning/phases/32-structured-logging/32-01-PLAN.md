# Phase 32-01 Plan: Structured Logging

## Objective

Implement structured logging with consistent field names (OBS-33), configurable log format (OBS-34), per-component log levels (OBS-37), and Python log forwarding (OBS-38).

## Tasks

### 1. Update ObservabilityConfig with per-component log levels

Add `ComponentLogLevels` struct to `src/observability/config.rs`:
- `worker_loop: tracing::Level`
- `dispatcher: tracing::Level`
- `accumulator: tracing::Level`

### 2. Update src/logging.rs to use ObservabilityConfig

- Accept `ObservabilityConfig` in `Logger::init()`
- Apply log format from config (json/pretty/simple) via `tracing-subscriber::fmt::fmt()`
- Build `EnvFilter` from per-component log levels
- Initialize `tracing_log::LogTracer` for Python log forwarding (OBS-38)

### 3. Add structured log events to worker_loop

In `src/worker_pool/mod.rs`, add `log::info!` / `log::error!` at:
- **Invoke start**: handler_id, topic, partition, offset, mode
- **Invoke completion**: handler_id, topic, partition, offset, elapsed_ms
- **Invoke error**: handler_id, topic, partition, offset, error_type

Fields follow `kafpy.{component}.{operation}` convention with consistent field names:
- `handler_id`
- `topic`
- `partition`
- `offset`

### 4. Add tracing_log dependency

Add to `Cargo.toml`:
```toml
tracing-log = "0.2"
```

### 5. Wire Python log forwarding

In `src/logging.rs`, after subscriber init:
```rust
tracing_log::LogTracer::init().expect("Failed to set LogTracer");
```

- `src/logging.rs` — use ObservabilityConfig, per-component levels, LogTracer
- `src/worker_pool/mod.rs` — add structured log events at invoke start/complete/error
