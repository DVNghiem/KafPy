   - `base_timestamp_ms: i64`
   - `current_timestamp_ms: i64`
   - `throughput_delta_pct: f64` — percent change in throughput (positive = improvement)
   - `latency_p50_delta_pct: f64`

   - Verify scenario_names match (return error via `anyhow::bail` if not)
   - Compute delta percentages: `(current - base) / base * 100.0`
   - Define regression threshold: latency regression = delta > 5.0%, throughput regression = delta < -5.0%
   - Build `regressions` and `improvements` vectors with human-readable descriptions
   - Memory delta is absolute bytes, not percentage

3. Add `regression_threshold_pct: f64` config field to `OutputConfig` (default: 5.0)

Follow Rust error handling: use `anyhow::Result` and `anyhow::bail!` for validation errors.

   ```

   | Metric | Base | Current | Delta % |

   | Throughput | {base} | {current} | {delta_pct:+.2f}% |
   | Latency P99 | {base} | {current} | {delta_pct:+.2f}% |
   ...

   ## Regressions
   {regression items or "None"}

   ## Improvements
   {improvement items or "None"}
   ```
