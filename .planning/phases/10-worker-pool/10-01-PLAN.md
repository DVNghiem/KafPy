3. **`worker_loop` async fn** — signature:
```rust
async fn worker_loop(
    mut rx: mpsc::Receiver<OwnedMessage>,
    handler: Arc<PythonHandler>,
    executor: Arc<dyn Executor>,
    queue_manager: Arc<QueueManager>,
    worker_id: usize,
    shutdown_token: CancellationToken,
)
```

The worker_loop implements the `select!` pattern from the research:
- Main loop uses `tokio::select!` with two branches:
  - `Some(msg) = rx.recv()`, if `active_message.is_none()` — picks up a message
  - `_ = shutdown_token.cancelled()`, if `active_message.is_none()` — checks cancellation when idle
- Maintains `active_message: Option<OwnedMessage>` to track in-flight work
- When a message is picked up:
  - Creates `ExecutionContext::new(msg.topic.clone(), msg.partition, msg.offset, worker_id)`
  - Calls `handler.invoke(&ctx, msg.clone()).await` (EXEC-10: spawn_blocking used internally)
  - Calls `executor.execute(&ctx, &msg, &result)` and matches outcome
  - If `result.is_ok()` → calls `queue_manager.ack(&msg.topic, 1)` (EXEC-13)
- When `shutdown_token.cancelled()` AND `active_message.is_none()` → logs and breaks
- After processing a message, checks `shutdown_token.is_cancelled()` before looping back (finish in-flight first — EXEC-12)

Structured logging (EXEC-11) via `tracing`:
- Worker start: `tracing::info!(worker_id = worker_id, "worker started");`
- Message pickup: `tracing::trace!(worker_id = worker_id, topic = %msg.topic, partition = msg.partition, offset = msg.offset, "worker picked up message");`
- Success: `tracing::debug!(worker_id = worker_id, topic = %ctx.topic, partition = ctx.partition, offset = ctx.offset, "handler executed successfully");`
- Error: `tracing::warn!(worker_id = worker_id, topic = %ctx.topic, partition = ctx.partition, offset = ctx.offset, exception = %exception, "handler raised exception");`
- Worker stop: `tracing::info!(worker_id = worker_id, "worker stopped");`

- Creates `CancellationToken::new()`

- For each `worker_id` in `0..n_workers`:
  - Clones receiver from `receivers.get(worker_id)` (use `.expect("receivers count must match n_workers")`)
  - Clones `Arc<PythonHandler>`, `Arc<dyn Executor>`, `Arc<QueueManager>`
  - Clones the `CancellationToken`
  - Calls `join_set.spawn(worker_loop(rx, handler, executor, queue_manager, worker_id, token))`
- Returns `Self { join_set, shutdown_token }`

6. **`WorkerPool::run` async method** — runs all workers until shutdown:
```rust
pub async fn run(mut self) {

}

This is called by the consumer to wait for all workers to finish. The JoinSet::shutdown() aborts all tasks and waits for completion.

pub async fn shutdown(&mut self) {
    tracing::info!("initiating worker pool shutdown");
    self.shutdown_token.cancel();

    tracing::info!("worker pool shutdown complete");
}
```
- Uses `shutdown()` not `abort_all()` — waits for in-flight completion (EXEC-12)
