## Key Changes

### CustomConsumerContext (src/consumer/context.rs)

- Implements rdkafka `ClientContext` + `ConsumerContext` traits
- `pre_rebalance` with `Rebalance::Revoke`: For each revoked partition, calls `offset_tracker.highest_contiguous()` then `store_offset()` + `commit_consumer_state(Sync)`
- `post_rebalance` with `Rebalance::Assign`: For each assigned partition, calls `offset_tracker.committed_offset()` then `seek(committed+1)`
- `ClientContext` log forwarding to tracing
- `pause_state` Arc for tracking paused partitions

### ConsumerRunner Changes (src/consumer/runner.rs)

- `StreamConsumer<CustomConsumerContext>` replaces `StreamConsumer`
- `ConsumerRunner::new()` now accepts `Arc<OffsetTracker>`, `Arc<dyn DlqRouter>`, `Arc<SharedDlqProducer>`
- Creates `CustomConsumerContext::new()` and uses `create_with_context()`
- `pause_partition(topic, partition)` / `resume_partition(topic, partition)` methods for backpressure

### ConsumerConfigBuilder (src/consumer/config.rs)

- Default `partition_assignment_strategy` changed to `CooperativeSticky`

### RuntimeBuilder (src/runtime/builder.rs)

- Reordered creation: DLQ components created BEFORE OffsetTracker
- OffsetTracker created, then ConsumerRunner with CustomConsumerContext
- Runner wired to OffsetTracker via `set_runner()`
