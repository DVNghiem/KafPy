  <name>Task 1: Improve ConsumerError with structured fields</name>
  <files>src/consumer/error.rs</files>
  <action>
Replace stringly-typed ConsumerError variants with structured fields carrying actionable context:

1. **Subscription** - convert from `Subscription(String)` to:
   ```rust
   #[error("failed to subscribe to topics at broker '{broker}': {message}")]
   Subscription { broker: String, message: String }
   ```

2. **Receive** - convert from `Receive(String)` to:
   ```rust
   #[error("message receive error for handler '{handler}' after {timeout_ms}ms: {message}")]
   Receive { handler: String, timeout_ms: u64, message: String }
   ```

3. **Serialization** - convert from `Serialization(String)` to:
   ```rust
   #[error("deserialization failed for message at {topic}:{partition}@{offset}; bytes preview: {bytes_preview:?} — {source}")]
   Serialization { topic: String, partition: i32, offset: i64, bytes_preview: Vec<u8>, #[source] source: serde_json::Error }
   ```
   - Note: the source should be `serde_json::Error` for JSON, but since other deserializers may be used, consider using `Box<dyn std::error::Error + Send + Sync>` as the source type so it's flexible.

4. **Processing** - convert from `Processing(String)` to:
   ```rust
   #[error("processing error for message at {topic}:{partition}@{offset}: {message}")]
   Processing { topic: String, partition: i32, offset: i64, message: String }
   ```

**Important:** Add `#[source]` annotations for wrapped errors so the error chain is preserved. Use `Box<dyn std::error::Error + Send + Sync>` for the Serialization source since multiple deserializers (JSON, msgpack) may be used.

Ensure all new variants continue to derive Debug via thiserror (thiserror derives Debug automatically when all fields implement Debug).
