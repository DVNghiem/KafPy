<task type="auto">
  <name>Task 2: Implement 5 check functions with real validation logic</name>
  <files>src/benchmark/hardening.rs</files>
  <action>
Replace the placeholder bodies in the 5 check functions with real validation logic:

**check_backpressure (HARD-04):**
- Consumer honors queue_depth limits without message loss under saturated load.
- Check: `result.error_rate < 0.001` (less than 0.1% message loss is acceptable).
- If passed: details="Message loss rate within acceptable threshold (error_rate < 0.1%)", suggestions=empty.
- If failed: details=format!("Message loss rate too high: {:.2}% (expected < 0.1%)", result.error_rate * 100.0), suggestions=vec!["Increase queue_depth capacity".to_string(), "Reduce message production rate".to_string(), "Scale consumer instances".to_string()].

**check_memory_leak (HARD-05):**
- No significant heap growth (> 1MB delta) over sustained run.
- Check: `result.memory_delta_bytes.abs() < 1_048_576` (1 MB = 1024 * 1024).
- If passed: details="Memory delta within acceptable threshold", suggestions=empty.
- If failed: details=format!("Memory grew by {:.2} MB (expected < 1 MB)", result.memory_delta_bytes as f64 / 1_048_576.0), suggestions=vec!["Check for unclosed file handles or connections".to_string(), "Review buffer pool sizing".to_string(), "Profile heap allocations with heap_allocated metrics".to_string()].

**check_graceful_shutdown (HARD-06):**
- Pending messages processed and offsets committed before exit on SIGINT.
- Check: `result.total_messages == result.scenario_config.num_messages` (all messages processed).
- If passed: details="All messages processed before shutdown", suggestions=empty.
- If failed: details=format!("{} of {} messages processed", result.total_messages, result.scenario_config.num_messages), suggestions=vec!["Increase graceful_shutdown_timeout".to_string(), "Check for deadlocks in handler code".to_string(), "Verify offset commit before exit".to_string()].

**check_dlq_drain (HARD-07):**
- All failed messages delivered to DLQ topic after graceful shutdown.
- Check: `result.error_rate > 0.0 implies result.error_rate == result.scenario_config.failure_rate` (failed messages match expected failure rate — indicating DLQ routing is working).
- Alternative check: when failure_rate > 0, verify total_messages matches expected (num_messages - failed messages were DLQ-routed).
- If passed: details="Failed messages routed to DLQ (error_rate matches failure_rate config)", suggestions=empty.
- If failed: details=format!("DLQ routing may not be working correctly: error_rate={:.2}% but failure_rate={:.2}%", result.error_rate * 100.0, result.scenario_config.failure_rate * 100.0), suggestions=vec!["Verify DLQ topic is configured".to_string(), "Check DLQ producer is sending messages".to_string(), "Confirm DLQ router is being called on retry exhaustion".to_string()].

**check_retry_budget (HARD-08):**
- Messages exhaust retry budget and route to DLQ (not infinite retry loop).
- Check: `result.error_rate > 0.0 implies result.duration_ms < infinite_retry_threshold`.
- Alternative: verify that when error_rate > 0, duration_ms is bounded (no infinite loops). A message processing that retries forever would show very high duration_ms per message.
- If passed: details="Retry budget respected: no infinite retry loops detected", suggestions=empty.
- If failed: details="Possible infinite retry loop detected (high duration with errors)".to_string(), suggestions=vec!["Verify RetryPolicy.max_attempts is configured".to_string(), "Confirm RetryCoordinator signals DLQ on max_attempts exceeded".to_string(), "Check that non-retryable failures are classified correctly".to_string()].

Add test cases:
- `test_backpressure_check_passes_on_low_error_rate()` — error_rate = 0.0001 passes
- `test_backpressure_check_fails_on_high_error_rate()` — error_rate = 0.01 fails
- `test_memory_leak_check_passes_on_small_delta()` — memory_delta_bytes = 512_000 passes
- `test_memory_leak_check_fails_on_large_delta()` — memory_delta_bytes = 2_000_000 fails
- `test_graceful_shutdown_check_passes_when_all_messages_processed()` — total == config.num_messages passes
- `test_graceful_shutdown_check_fails_when_messages_lost()` — total < config.num_messages fails
- `test_dlq_drain_check_passes_when_error_rate_matches_failure_rate()` — error_rate == failure_rate passes
- `test_retry_budget_check_passes_on_bounded_duration()` — normal duration passes

Use the BenchmarkResult and ScenarioConfig types already imported from results.rs.
</action>
