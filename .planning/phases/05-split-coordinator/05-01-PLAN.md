# Key semantics to preserve (from offset_tracker.rs):
#
# 1. highest-contiguous-offset algorithm:
#    - PartitionState.committed_offset starts at -1
#    - ack(): insert offset, then while pending_offsets.contains(committed_offset + 1),
#      remove from pending and increment committed_offset
#    - highest_contiguous() returns committed_offset
#
# 2. has_terminal gating:
#    - should_commit() returns false when has_terminal=true (per-partition only)
#    - set_terminal() is idempotent (set once, never cleared)
#    - D-05: FailureCategory::Terminal sets has_terminal=true
#
# 3. Two-phase contract:
#    - store_offset must precede commit
#    - flush_failed_to_dlq() called BEFORE graceful_shutdown()
</context>
