    1. Add `use crate::routing::Router;` import
    2. Add `router: Option<Arc<dyn Router>>` field to `ConsumerDispatcher` struct
    3. Add `dlq_producer: Option<SharedDlqProducer>` field to `ConsumerDispatcher` struct
    4. Add `pub fn new_with_router(runner: ConsumerRunner, router: Arc<dyn Router>, dlq_producer: SharedDlqProducer) -> Self` constructor
    5. Keep `new()` for backward compatibility (router = None, dlq_producer = None — no routing, existing behavior)
    6. In `run()` loop, before `dispatcher.send_with_policy_and_signal`:
       - Build `RoutingContext::from_message(&msg)` (zero-copy borrow)
       - If `self.router.is_some()`: call `router.route(ctx)` to get `RoutingDecision`
       - Match on decision:
         - **Route(handler_id)**: Set `msg.handler_id = Some(handler_id.clone())`, then call `dispatcher.send_to_handler(&handler_id, msg)` (bypasses topic lookup)
         - **Drop**: Do NOT dispatch. Commit offset for this message directly (since no handler processed it, advance the consumer offset). Store offset via `runner.store_offset(msg.topic, msg.partition, msg.offset)`.
         - **Reject(reason)**: Build `DlqMetadata::new(...)` with original_topic, original_partition, original_offset, failure_reason=reason.to_string(), attempt_count=1, timestamps. Call `dlq_producer.produce_async(...)`. Commit offset after DLQ routing.
         - **Defer**: Proceed with normal `dispatcher.send_with_policy_and_signal(msg, policy)` — topic-based routing as today
       - For Drop and Reject paths: call `runner.store_offset(...)` then `runner.commit()` to advance offset
    7. Ensure Drop path does NOT call any handler channel send — message is filtered silently
    8. Ensure Reject path produces to DLQ before offset commit
  </action>

    <automated>cargo check --lib 2>&1 | head -30</automated>
  </verify>
  <done>ConsumerDispatcher calls router.route() before queue selection; Drop advances offset without handler call; Reject produces to DLQ with metadata; Defer does normal topic dispatch; Route routes to handler-named queue via send_to_handler</done>
