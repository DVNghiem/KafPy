**The fix**: Modify `send_with_policy` to also return the `BackpressureAction` signal when backpressure occurs, without changing the public `Result` signature.

```rust
    // ... existing send_with_policy logic, but track the action for FuturePausePartition
    // Returns (Err(DispatchError::Backpressure(topic)), Some(FuturePausePartition(topic))) 
    // for that case instead of just (Err(...), None)
}
```

Now update `ConsumerDispatcher::run()` to call this new method and handle the signal:

```rust
pub async fn run(&amp;self, policy: &amp;dyn BackpressurePolicy) {
    let mut stream = self.runner.stream();
    while let Some(result) = stream.next().await {
        match result {
            Ok(msg) =&gt; {
                let topic = msg.topic.clone();
                let (outcome, pause_signal) = self.dispatcher.send_with_policy_and_signal(msg, policy);
                match outcome {
                    Ok(outcome) =&gt; {
                        // Check if resume is needed
                        self.check_resume(&amp;topic, outcome.queue_depth);
                    }
                    Err(DispatchError::Backpressure(_)) =&gt; {
                        // If pause signal present, trigger actual pause
                        if let Some(BackpressureAction::FuturePausePartition(pause_topic)) = pause_signal {
                            if let Err(e) = self.pause_partition(&amp;pause_topic) {
                                tracing::error!("failed to pause topic '{}': {}", pause_topic, e);
                            } else {
                                tracing::warn!("paused topic '{}' due to backpressure", pause_topic);
                                self.paused_topics.lock().insert(pause_topic.clone());
                            }
                        }
                    }
                    Err(DispatchError::HandlerNotRegistered(topic)) =&gt; {
                        tracing::debug!("no handler for topic '{}', skipping", topic);
                    }
                    Err(e) =&gt; {
                        tracing::error!("dispatch error: {}", e);
                    }
                }
            }
            Err(e) =&gt; {
                tracing::error!("consumer error: {}", e);
            }
        }
    }
}
```

Then add the `pause_partition()` and `resume_partition()` methods to `ConsumerDispatcher`:

```rust
/// Pauses consumption for all partitions of `topic` using rdkafka pause().
fn pause_partition(&amp;self, topic: &amp;str) -&gt; Result&lt;(), ConsumerError&gt; {
    let handles = self.partition_handles.lock();
    if let Some(tpl) = handles.get(topic) {
        // StreamConsumer::pause takes &TopicPartitionList
        self.runner.consumer.pause(tpl)
    } else {
        Err(ConsumerError::Subscription(format!(
            "no partition handle for topic '{}' - has the consumer been assigned partitions?",
            topic
        )))
    }
}

/// Resumes consumption for all partitions of `topic` using rdkafka resume().
fn resume_partition(&amp;self, topic: &amp;str) -&gt; Result&lt;(), ConsumerError&gt; {
    let handles = self.partition_handles.lock();
    if let Some(tpl) = handles.get(topic) {
        self.runner.consumer.resume(tpl)
    } else {
        Err(ConsumerError::Subscription(format!(
            "no partition handle for topic '{}'",
            topic
        )))
    }
}
```

Also update `check_resume()` to actually call `resume_partition()`:

```rust
fn check_resume(&amp;self, topic: &amp;str, current_depth: usize) {
    let Some(capacity) = self.dispatcher.get_capacity(topic) else { return };
    let threshold = (capacity as f64 * self.resume_threshold) as usize;

    if current_depth &lt; threshold {
        if self.paused_topics.lock().remove(topic) {
            if let Err(e) = self.resume_partition(topic) {
                tracing::error!("failed to resume topic '{}': {}", topic, e);
            } else {
                tracing::info!("resumed topic '{}' (depth {} &lt; threshold {})", topic, current_depth, threshold);
            }
        }
    }
}
```

**DISP-18 requirement**: The `FuturePausePartition` action now triggers actual `consumer.pause()` via rdkafka, and resume happens when queue drains below threshold.

Update `QueueManager::send_to_handler` to use `try_acquire_semaphore()` BEFORE `try_send`.

The semaphore should be acquired BEFORE incrementing inflight and before `try_send`. If the semaphore cannot be acquired (try_acquire returns false), return `Err(DispatchError::Backpressure(topic))` immediately without modifying any counters.

```rust
// In send_to_handler method:
pub(crate) fn send_to_handler(
    &amp;self,
    message: OwnedMessage,
) -&gt; Result&lt;crate::dispatcher::DispatchOutcome, DispatchError&gt; {
    let topic = message.topic.clone();
    let partition = message.partition;
    let offset = message.offset;

    let guard = self.handlers.lock();
    let entry = guard
        .get(&amp;topic)
        .ok_or_else(|| DispatchError::HandlerNotRegistered(topic.clone()))?;

    // DISP-15: Try to acquire semaphore permit BEFORE any counter modification
    if !entry.metadata.try_acquire_semaphore() {
        return Err(DispatchError::Backpressure(topic));
    }

    match entry.sender.try_send(message) {
        Ok(()) =&gt; {
            entry.metadata.inc_queue_depth();
            entry.metadata.inc_inflight();
            Ok(crate::dispatcher::DispatchOutcome {
                topic,
                partition,
                offset,
                queue_depth: entry.metadata.get_queue_depth(),
            })
        }
        Err(TrySendError::Full(_)) =&gt; Err(DispatchError::QueueFull(topic)),
        Err(TrySendError::Closed(_)) =&gt; Err(DispatchError::QueueClosed(topic)),
    }
}
```

The `try_acquire_semaphore()` method on `HandlerMetadata` should be:

```rust
/// Attempts to acquire a semaphore permit if one is configured.
/// Returns true if permit acquired or no semaphore configured.
/// Returns false if semaphore is configured but no permits available.
/// Non-blocking.
pub(crate) fn try_acquire_semaphore(&amp;self) -&gt; bool {
    if let Some(sem) = &amp;self.semaphore {
        sem.try_acquire().is_ok()
    } else {
        true  // No semaphore = unlimited
    }
}
```

**Important**: The semaphore permit must be released when Python calls `ack()`. Since `ack()` already decrements `inflight` and `queue_depth`, we need to also release the semaphore permit there.

Add a `release_semaphore_permit(&amp;self, count: usize)` method to `HandlerMetadata` that stores permits to release. But since `Semaphore` permits are not countable in this way, we need a different approach.

**Alternative design**: The semaphore is used to limit CONCURRENT handler invocations. Each message dispatch "holds" a permit until `ack()`. Since we can't easily "return" multiple permits, we track permits separately:

Add `outstanding_permits: AtomicUsize` to `HandlerMetadata`. When `try_acquire_semaphore()` succeeds, increment `outstanding_permits`. When `ack()` is called, decrement `outstanding_permits` by `count` and release that many permits back to the semaphore.

```rust
// HandlerMetadata:
pub(crate) semaphore: Option&lt;Arc&lt;Semaphore&gt;&gt;,
pub(crate) outstanding_permits: AtomicUsize,  // NEW

// try_acquire_semaphore:
if let Some(sem) = &amp;self.semaphore {
    if sem.try_acquire().is_ok() {
        self.outstanding_permits.fetch_add(1, Ordering::Relaxed);
        true
    } else {
        false
    }
} else {
    true
}

// New method release_permits(count):
pub(crate) fn release_permits(&amp;self, count: usize) {
    if let Some(sem) = &amp;self.semaphore {
        let current = self.outstanding_permits.load(Ordering::Relaxed);
        let to_release = count.min(current);
        for _ in 0..to_release {
            sem.add_permit();
        }
        self.outstanding_permits.fetch_sub(to_release, Ordering::Relaxed);
    }
}
```

**Actually**, Tokio `Semaphore` has `add_permits(n)` method (not `add_permit()` - that's wrong). Use `sem.add_permits(count)` to release multiple permits at once.

Wait, looking at tokio docs: `Semaphore` has `add_permits(n)` to add permits, and `try_acquire()` returns a `Permit` that auto-releases when dropped. The cleanest approach is to NOT manually track - instead, the `Permit` is tied to the message. When `ack()` is called, we drop the implicit permit...

Actually this is getting complex. Let me use a simpler approach: since `try_acquire()` returns a `Permit` that holds the permit until dropped, we can't easily store it across the async boundary (Python ack happens from Python, not Rust).

**Simpler design for DISP-15**: Track outstanding permits count and call `sem.add_permits(count)` on `ack()`. This is correct - we acquired `count` permits over time, now we release them.

```rust
// In HandlerMetadata:
pub(crate) fn try_acquire(&amp;self) -&gt; bool {
    if let Some(sem) = &amp;self.semaphore {
        match sem.try_acquire() {
            Ok(permit) =&gt; {
                // Hold the permit - we'll release it on ack
                // Actually we can't store permit across await points
                // Use a different approach: just track count
                true
            }
            Err(_) =&gt; false,
        }
    } else {
        true
    }
}
```

Let me reconsider. The cleanest design:
1. `try_acquire()` just does `sem.try_acquire().is_ok()` - non-blocking
2. If it succeeds, we increment `outstanding_permits` counter
3. When Python calls `ack(count)`, we decrement `outstanding_permits` and call `sem.add_permits(count)`

This is correct because we're just tracking how many permits we've given out and returning them when ack comes in.

**DISP-15 requirement**: Non-blocking `try_acquire()` returns `Backpressure` if no permit available. The consumer loop continues - it does NOT wait for a permit.
